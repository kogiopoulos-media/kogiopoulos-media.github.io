Step-by-step instructions

Step 1

Create a GitHub account

  1. Go to github.com and click Sign up
  2. Choose a username — this will appear in your site URL, e.g. kogiopoulosmedia
  3. Verify your email address
Step 2

Create a new repository

  1. Once logged in, click the + icon in the top right → New repository
  2. Name it exactly: kogiopoulosmedia.github.io
    (Replace kogiopoulosmedia with your exact GitHub username)
  3. Set it to Public
  4. Leave everything else as default and click Create repository
⚠️ The repository name must match your username exactly. If your username is jsmith, the repo must be named jsmith.github.io
Step 3

Unzip and upload your files

  1. Unzip the kogiopoulos-media.zip file on your computer
  2. On your new GitHub repository page, click uploading an existing file (or drag files in)
  3. Select all files inside the unzipped folder and drag them into the GitHub upload area
  4. Wait for all files to upload — this may take a minute due to the photo file sizes
  5. Scroll down, add a commit message like Initial site upload, and click Commit changes
📁 Make sure you upload the contents of the folder, not the folder itself. GitHub should show all HTML files and JPG images at the root level — not inside a subfolder.
Step 4

Enable GitHub Pages

  1. In your repository, click Settings (top menu bar)
  2. In the left sidebar, click Pages
  3. Under Branch, select main from the dropdown
  4. Leave the folder as / (root)
  5. Click Save
Step 5

Visit your live site

After a minute or two, your site will be live at:

https://YOUR-USERNAME.github.io

GitHub Pages can take 2–5 minutes to deploy after saving. Refresh the Settings → Pages screen until you see a green "Your site is live" message with the URL.

Using a custom domain (optional)

If you own a domain like kogiopoulosmedia.com, you can point it to your GitHub Pages site for free.

  1. In Settings → Pages, enter your domain name in the Custom domain field and click Save
  2. Log into your domain registrar (e.g. GoDaddy, Namecheap, Squarespace Domains)
  3. Add the following A records pointing to GitHub's servers:
185.199.108.153
185.199.109.153
185.199.110.153
185.199.111.153
  1. Also add a CNAME record: www → YOUR-USERNAME.github.io
  2. DNS changes can take up to 48 hours to propagate
  3. Once live, tick Enforce HTTPS in GitHub Pages settings

Updating your site in the future

To update content, simply go to your repository on GitHub, click the file you want to edit or upload a new version, and commit the change. The site will update within a minute or two.

To add new photos to the portfolio, upload the JPG files to the repository and update the relevant HTML file to reference them.

Need help?

GitHub has excellent documentation at docs.github.com/en/pages. If something isn't working, the most common fix is to double-check that all files are at the root level of the repository (not inside a subfolder).
